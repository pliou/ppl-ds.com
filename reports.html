Magento report modules

PPL Reports

A group of Magento report modules using the same admin shell. Reports can be installed separately and share cards, tables, CSV exports and scheduled emails.

Report Core Admin reports CSV exports Custom software license

Main report modules

Each module provides one report or report function. Report Core provides the shared admin pages.

Report Core

Admin menu, routes, ACL, registry, shell UI, shared table/card rendering, CSV support and scheduled report emails.

Revenue

Revenue report for monitoring sales performance over time.

Coupon Usage

Coupon report for campaign usage, discounts and promotion impact.

Orders by Hour

Hourly order distribution for staffing, shipping checks and traffic analysis.

Payment Revenue

Revenue split by payment method for payment performance review.

Product Data Quality

Catalog data checks for product content and merchandising readiness.

Dashboard Cards

Small report cards for Magento admin dashboards.

Recent Orders Export

CSV export for recent orders.

Refund Rate

Refund rate report for identifying return pressure and product or process issues.

Search Zero Results

Table report for search terms that returned no products.

Stale Products

Report for products that need review because activity or freshness signals are weak.

Top SKUs

SKU performance report for identifying key products and changing demand.

Admin Daily Digest

Email digest with selected report values.

Featured report extension

PPL Report Top SKUs

Top SKUs gives Magento teams a focused view of the products that drive orders, quantity, revenue and average order value in the selected PPL Reports period.

PPL Hub > PPL Reports CSV export Action Widget card Magento 2.4

What it answers

Which SKUs matter most right now, whether the signal comes from many orders, high quantity or larger revenue per order.

Where it helps

Purchasing, inventory checks, merchandising, campaign review and product content priorities all get a faster starting point.

How it fits

The report uses PPL Report Core for period handling, table rendering, ACL behavior and CSV export, excludes parent order items, and can surface on the Action Widget board.

Revenue ranking

The main report highlights the highest-revenue SKUs for the selected period, with a bar chart for quick scanning.

Exact metrics

The table complements the chart with SKU, orders, quantity, revenue and AOV so teams can compare product behavior directly.

Widget board

A compact card can sit beside other urgent PPL reports for regular admin review.

Marketplace visual

The extension has a product-level visual for marketplace and documentation pages.

Performance story

A second marketplace visual communicates the broader performance-reporting use case.

Why it is grouped

Reports share the same rendering layer and scheduling model. Stores can install the report modules they need.

Admin path: PPL Hub > PPL Reports. If no report modules are installed, Report Core shows an empty state.
